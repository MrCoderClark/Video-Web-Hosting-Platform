# Phase 2: Video Upload & Storage

⚠️ **Medium** — Multiple moving parts (frontend dropzone, backend endpoint, Supabase Storage, DB)

## Overview
Implement drag-and-drop video upload with progress tracking, store files in Supabase Storage, and create the `videos` table with RLS policies.

## Tasks

### 2.1 DB Migration — Videos Table
- Create `supabase/migrations/002_videos.sql`:
  - `videos` table: id, user_id, title, description, status, original_filename, original_size_bytes, storage_path, thumbnail_url, duration_seconds, created_at, updated_at
  - Status enum: `uploading`, `queued`, `processing`, `ready`, `error`
  - RLS: users can CRUD their own videos, anyone can SELECT public videos
  - Index on user_id, status, created_at

### 2.2 Backend — Upload Endpoint
- `POST /api/videos/upload` — accepts multipart file upload
  - Validate file type (mp4, mov, webm, avi, mkv)
  - Validate file size (configurable max, default 2GB)
  - Stream file to Supabase Storage bucket `videos`
  - Create row in `videos` table with status `queued`
  - Return video ID + metadata
- `GET /api/videos` — list current user's videos
- `GET /api/videos/:id` — get single video details
- Create `routers/videos.py` and `services/storage.py`

### 2.3 Supabase Storage Setup
- Create `videos` bucket (via migration or manual setup)
- Storage policies: authenticated users can upload to their own folder
- Path convention: `{user_id}/{video_id}/{original_filename}`

### 2.4 Frontend — Upload Page
- Drag-and-drop dropzone component with file type validation
- File preview (name, size, type) before upload
- Upload progress bar (percentage + bytes transferred)
- Title/description form fields (editable before and during upload)
- Post-upload: redirect to video detail or dashboard
- Error handling with toast notifications

### 2.5 Frontend — API Client
- Create `src/lib/api.ts` — axios/fetch wrapper with auth token injection
- Upload function with XMLHttpRequest for progress tracking (or fetch + ReadableStream)

## Validation Tests
1. Drag a .mp4 file onto dropzone → file is accepted, preview shows
2. Click upload → progress bar fills → file appears in Supabase Storage
3. After upload → row in `videos` table with status `queued`, correct user_id
4. `GET /api/videos` returns user's videos only (RLS)
5. Upload a .txt file → rejected with error message
6. Upload without auth → 401
7. Large file (>100MB) uploads successfully with progress
