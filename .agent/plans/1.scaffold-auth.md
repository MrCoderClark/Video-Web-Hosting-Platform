# Phase 1: Scaffold & Auth

✅ **Simple** — Single-pass executable, low risk

## Overview
Initialize both frontend and backend apps, connect to self-hosted Supabase, and implement full auth flow.

## Tasks

### 1.1 Frontend — Next.js Setup
- `npx create-next-app@latest frontend --typescript --tailwind --eslint --app --src-dir`
- Install shadcn/ui: `npx shadcn@latest init`
- Install dependencies: `@supabase/supabase-js`, `@supabase/ssr`, `lucide-react`
- Configure `next.config.ts` (image domains for Supabase)
- Create `src/lib/supabase.ts` (browser + server clients)

### 1.2 Backend — FastAPI Setup
- Create `backend/` directory
- Create `requirements.txt`: fastapi, uvicorn, pydantic-settings, asyncpg, supabase, python-multipart, python-jose
- Create venv: `python -m venv venv`
- Create `main.py` with CORS, health endpoint
- Create `config.py` with Pydantic Settings (env vars)
- Create `database.py` with asyncpg pool (Supavisor connection)
- Create `auth.py` — verify Supabase JWT from Authorization header

### 1.3 Supabase Schema
- Create `supabase/migrations/001_profiles.sql`:
  - `profiles` table (id, display_name, avatar_url, created_at)
  - RLS policy: users read/update own profile only
  - Trigger: auto-create profile on auth.users insert

### 1.4 Auth Flow — Frontend
- `/login` page — email/password form, Supabase signInWithPassword
- `/signup` page — email/password form, Supabase signUp
- Navbar with user state (logged in / logged out)
- Middleware for protected routes (`/upload`, `/dashboard`, `/settings`)
- `src/hooks/use-auth.ts` — auth state hook with Supabase onAuthStateChange

### 1.5 Auth Flow — Backend
- `GET /api/health` — public health check
- `GET /api/me` — return current user profile (requires valid JWT)
- Dependency injection: `get_current_user()` extracts + verifies JWT

### 1.6 Environment & Config
- Create `.env.example` with all required variables
- Create `.gitignore` (node_modules, venv, .env, __pycache__, .next, etc.)

## Validation Tests
1. `npm run dev` — Next.js serves on http://localhost:3000
2. `uvicorn main:app --reload --port 8001` — FastAPI serves, `/api/health` returns 200
3. Navigate to `/signup` → create account → redirected to home
4. Navigate to `/login` → sign in → navbar shows user
5. `/api/me` with valid token returns profile
6. `/api/me` without token returns 401
7. Direct DB query: profile row exists after signup
8. RLS test: user A cannot query user B's profile
